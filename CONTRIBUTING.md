# Contributing to Dictionary Lens

First off, thank you for considering contributing to **Dictionary Lens**! Your support and contributions are what make open-source projects thrive. We appreciate any help you can provide, whether it’s bug fixes, feature development, or improving documentation.

## How Can You Contribute?

### 1. Reporting Bugs
If you come across a bug, please create an issue with:
- A clear and descriptive title.
- Detailed steps to reproduce the bug.
- The expected behavior and what actually happens.
- Any screenshots, logs, or error messages that help illustrate the problem.

### 2. Suggesting Enhancements
If you have ideas to improve the app, feel free to open an issue for discussion. Provide as much detail as possible:
- A clear description of the enhancement.
- Any potential benefits or challenges you foresee.
- Possible solutions or alternatives.

### 3. Submitting Code Changes
To contribute code:
1. Fork the repository to create a copy under your own GitHub account.
2. Clone your fork to your local machine.
3. Create a new branch for your feature or bug fix.
4. Make your changes and commit them with clear and descriptive commit messages.
5. Push your branch to your forked repository.
6. Submit a pull request with details about your changes.

### 4. Code Style Guidelines
- Follow **Quasar's and Vue.js's coding standards** for the frontend.
- Use **Kotlin best practices** for native code.
- Run linting and formatting tools to ensure your code adheres to project rules.

### 5. Review Process
- PRs will be reviewed by project maintainers.
- If changes are required, maintainers will provide feedback and request updates.
- Once approved, your PR will be merged into the main branch.

## Getting Started
Make sure to follow these steps to get the project up and running on your local machine:
1. Install dependencies using yarn or npm.
2. Run the development server using the Quasar CLI.

## Future Development
There may be work in the future involving server-side development using **PHP** and **Laravel** for handling user authentication and other backend functionalities. Stay tuned for updates!

## Code of Conduct
This project follows a **Code of Conduct** to create a welcoming and inclusive environment for all contributors. By participating, you agree to uphold this standard.

Thank you for your contributions and support! Together, we can make **Dictionary Lens** a valuable tool for users worldwide.
